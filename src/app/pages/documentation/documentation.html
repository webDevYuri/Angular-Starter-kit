Architecture

Folder Structure

Feature-based architecture for scalability and maintainability.

Project Overview

The full src/app/ directory at a glance.

src/app/
|-- core/               # Core functionality
|   |-- constants/      # App-wide constants & config
|   |-- data/           # Static/shared data sources
|   |-- directives/     # Custom attribute directives
|   |-- enums/          # Named constant sets
|   |-- guards/         # Route guards
|   |-- interceptors/   # HTTP interceptors
|   |-- interfaces/     # TypeScript interfaces
|   |-- models/         # Classes with behavior
|   |-- pipes/          # Custom pipes
|   |-- services/       # Singleton services
|   |-- utils/          # Pure helper functions
|   \-- validators/     # Custom form validators
|-- layouts/            # Layout components
|-- pages/              # Page-level components
|-- sections/           # Reusable section components
|-- shared/             # Shared components & utilities
|-- app.routes.ts       # Route definitions
|-- app.ts              # Root component
\-- app.html            # Root template
core/

Application-wide singletons and logic.

Contains application-wide singletons and logic that should only be instantiated once. Nothing in core should be imported by shared or feature modules directly.

layouts/

Shared UI shells for pages.

Layout components wrap pages with shared UI elements like headers, sidebars, and footers. Pages are rendered inside the layout's <router-outlet />.

layouts/
|-- main-layout/           # Header + content area
|   |-- main-layout.ts
|   |-- main-layout.html
|   \-- main-layout.css
|-- admin-layout/          # Sidebar + topbar for admin
\-- auth-layout/           # Centered card layout for auth

Usage: Wrap child routes with a layout in app.routes.ts by setting the layout as the parent component and pages as children.

pages/

Route-level components.

Page-level components that map directly to routes. Each page gets its own folder with its component, template, and styles.

pages/
|-- home/                  # Landing page (/)
|   |-- home.ts
|   |-- home.html
|   \-- home.css
|-- documentation/         # Documentation (/documentation)
|-- dashboard/             # Dashboard (/dashboard)
|-- login/                 # Login (/login)
\-- not-found/             # 404 page (/**)

Convention: One page = one route. Keep pages thin - move complex UI blocks into sections/ and reusable pieces into shared/.

sections/

Larger page-specific UI blocks - hero banners, feature grids, pricing tables.

sections/
|-- hero-banner/
|-- feature-grid/
|-- pricing-table/
\-- testimonials/
shared/

Small generic components reused across multiple pages.

shared/
|-- toast/
|-- confirm-modal/
|-- phone-input/
\-- date-picker/

Sections vs Shared: Sections are larger, page-specific blocks. Shared components are smaller, generic pieces (buttons, modals, inputs) reused everywhere.

Service

Toast Notifications

Simple, non-blocking alerts that appear at the top of the screen.

Available Methods

{{ method.call }}

{{ method.desc }}

Example Usage

import { ToastService } from './core/services/toast.service';

export class MyComponent {
  constructor(private toastService: ToastService) {}

  handleSuccess() {
    this.toastService.success('Operation successful!');
  }

  handleError() {
    this.toastService.error('Something went wrong');
  }
}

Tip: Toasts automatically disappear after 3 seconds and are perfect for quick feedback after user actions.

Service

Confirmation Modal

Blocking dialogs for important user confirmations.

Modal Types

{{ type.name }}

{{ type.desc }}

Example Usage

import { ConfirmService } from './core/services/confirm.service';

export class MyComponent {
  constructor(private confirmService: ConfirmService) {}

  async deleteItem() {
    const confirmed = await this.confirmService.confirm({
      title: 'Delete Item',
      message: 'This action cannot be undone. Continue?',
      confirmText: 'Delete',
      cancelText: 'Cancel',
      type: 'danger'
    });

    if (confirmed) {
      // proceed with deletion
    }
  }
}

Note: The confirm() method returns a Promise that resolves to true if confirmed, false if cancelled.

Available Animations

Usage in Templates

<div data-aos="fade-up">
  Animates on scroll
</div>

<!-- With delay -->
<div data-aos="fade-up" data-aos-delay="200">
  Delayed animation
</div>

<!-- With custom duration -->
<div data-aos="fade-up" data-aos-duration="800">
  Slower animation
</div>

Global Configuration

Initialized in app.ts with these defaults.

import AOS from 'aos';

AOS.init({
  duration: 600,      // Animation duration in ms
  easing: 'ease-out', // Easing function
  once: true          // Animate only once per element
});

Dynamic Content

Refreshing AOS after *ngIf toggles.

import AOS from 'aos';

toggleContent() {
  this.showContent = !this.showContent;
  setTimeout(() => AOS.refresh(), 50);
}

Tip: Use staggered data-aos-delay values (100, 200, 300) on sibling elements to create a cascading entrance effect.
